# Open Petition to QNX Software Systems and BlackBerry: Request to Re-License the 2009 QNX Neutrino Source Code Under Apache License 2.0

To the leadership and engineering teams of QNX Software Systems and
BlackBerry Limited,

We, the undersigned developers, researchers, engineers, and
operating-system enthusiasts, respectfully request that the historical
source code release of *QNX Neutrino RTOS (circa 2007–2009)* be relicensed
under the *Apache License 2.0* or a similarly permissive open-source
license approved by the Open Source Initiative.

## Background

The QNX operating system has long been recognized as one of the most
elegant and influential microkernel designs ever deployed in production
systems. Its architecture — centered on synchronous message passing and the
*send–receive–reply* IPC model — has shaped generations of engineers working
in real-time systems, embedded devices, and safety-critical software.

For many years, QNX made portions of its source code available through the
QNX community program. This program allowed developers and students to
study the system and build it, but the licensing terms limited
redistribution and community-driven evolution.

As a result, the code has remained historically important yet largely
inaccessible to the broader open-source ecosystem.

## Why Relicensing Would Be Valuable

Releasing the historical QNX Neutrino code under a modern permissive
license such as Apache 2.0 would create several positive outcomes:

### 1. Preservation of an important operating-system architecture

QNX represents a major milestone in microkernel design and real-time
operating systems. An open license would ensure that this work can continue
to be studied, taught, and improved by future generations.

### 2. Academic and research benefits

Universities often teach operating-system concepts using open kernels.
Making this code available under an OSI-approved license would allow
researchers and students to examine a mature production microkernel rather
than simplified teaching systems.

### 3. Ecosystem goodwill

Many companies have strengthened their developer ecosystems by
open-sourcing historical or foundational components. A relicensing effort
would demonstrate continued leadership in systems engineering and reinforce
QNX's reputation for technical excellence.

### 4. No conflict with current commercial offerings

The requested release concerns a historical code snapshot from roughly
2009. Modern QNX products have evolved significantly since then, and
open-sourcing this version would not diminish the value of current
commercial products, support contracts, or safety-certified releases.

### 5. Encouraging experimentation and innovation

Developers interested in message-passing operating systems often attempt to
recreate QNX-style architectures from scratch. Providing access to the
historical code would enable better experimentation while ensuring proper
attribution to the original engineering work.

## Why Apache License 2.0

The Apache License 2.0 is widely respected across industry and academia
because it:

- provides clear patent protection,
- is business-friendly and permissive,
- encourages both commercial and non-commercial collaboration.

These qualities make it particularly suitable for embedded and
systems-level software.

## Our Request

We respectfully ask QNX Software Systems and BlackBerry to consider:

1. Releasing the 2007–2009 QNX Neutrino source code under the Apache
   License 2.0 or another OSI-approved permissive license.
2. Publishing the code in a publicly accessible repository for historical
   preservation and study.
3. Allowing the community to maintain derivative educational or
   experimental versions.

## Closing

QNX has played a crucial role in the history of real-time and embedded
operating systems. Opening the historical source code would preserve that
legacy and inspire future generations of systems engineers.

We greatly appreciate the engineering excellence that has gone into QNX
over the decades, and we hope that this legacy can continue to benefit the
broader software community.
